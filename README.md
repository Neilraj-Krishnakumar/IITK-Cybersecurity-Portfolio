Breakdown of the 7 Layers

Project 1: mfa_gate_final.py (MFA Entry Gate)
- Why I made it: To create a perimeter wall that blocks bad bots and automated script requests.
- How it works: It asks for username and credentials and then makes a random math calculation challenge that you have to solve to pass proxy checks.
- The Upgrade: First it was a simple password check on my phone's PyDroid app, but on my laptop I upgraded it into a DevSecOps Multi-Factor gate loop layout.
- Output Proof: mfa_output.png

Project 2: traffic_sniffer.py (Packet Interceptor Node)
- Why I made it: To simulate live network monitoring and inspect packet data streams inside VS Code.
- How it works: It grabs live network protocol data (TCP, UDP, HTTPS) and extracts the source and destination IP blocks.
- Design Details: I injected Burp Suite sensor logging frames to match professional TryHackMe lab testing concepts.
- Output Proof: sniffer_output.png

Project 3: forensic_scraper.py (Incident Log Auditor)
- Why I made it: To do post incident data scraping inside system data logs after a breach.
- How it works: Loops through system records and flags compromised indicators instantly.
- Design Details: I mixed my custom state choosing arrays with OWASP Juice Shop validation rules to catch malicious SQL injection vectors.
- Output Proof: forensics_output.png

Project 4: crypto_vault.py (Symmetric Key Cipher)
- Why I made it: To keep confidential numbers safe during transit or cloud storage.
- How it works: Takes random numbers and passes them through a while loop shift index mechanism.
- Design Details: Built completely on my laptop as a dynamic symmetric key transformation cipher layer.
- Output Proof: crypto_output.png

Project 5: auth_brute_force.py (Sentinel Access Watcher)
- Why I made it: To defend portal screens against high velocity brute force dictionary scans.
- How it works: Tests keys from a password pool and activates a lockdown if threshold is crossed.
- Design Details: Added Burp Suite sentinel rules and Antigravity 2.0 loop boundaries to block the firewall endpoint instantly.
- Output Proof: brute_output.png

Project 6: password_evaluator.py (Complexity Governance Auditor)
- Why I made it: To enforce password complexity across profiles.
- How it works: Checks the character length of access string variables and rejects weak targets.
- Design Details: Written on laptop as a strict enterprise security compliance test based on SonarQube quality gate standards.
- Output Proof: password_evaluator_output.png

Project 7: network_sentinel.py (Core Diagnostics Node)
- Why I made it: To give a master control dashboard showing overall health of the ecosystem.
- How it works: Pings node arrays (Gateway, Firewall) and measures response speed latencies.
- Design Details: Added a DevSecOps microsecond telemetry tracker inside VS Code. I explicitly signed this layer with my full name (Neilraj Krishna Kumar) right in the console output string to certify it.
- Output Proof: network_sentinel_output.png
